Trilium Notes 是什么、能做什么、怎么用

Trilium Notes 是 zadam开发的一款开源、跨平台的「分层笔记」应用，可以构建无限节点的知识库笔记，支持双链笔记，自动生成知识笔记地图，前端主题页面可以高度自由定制等，既可以当单机笔记使用，也能自己搭建服务器部署同步。

一、核心定位

  • 树状层级：笔记无限嵌套，像文件夹一样管理；支持“克隆”功能，同一条笔记可镜像到多处

  • 本地优先：数据默认存在本地 SQLite，支持整库或单笔记 AES-256 加密

  • 自托管：可一键启动内置服务器，浏览器即可远程访问；也提供 Docker等部署方案

二、主要功能亮点

  • 编辑器：所见即所得 + Markdown 即时预览，Nriver大佬的汉化版集成了内嵌 Excalidraw 手绘、LaTeX 公式、代码高亮

  • 导航与检索：全文/属性搜索、关系图谱、链接图、note hoisting（聚焦单分支）

  • 自动化：支持 JavaScript 脚本、REST API，可做模板、批量操作、外部集成

  • 版本控制：每次修改自动生成历史快照，可随时回滚

  • 导入导出：笔记批量迁移无压力

  • 分享笔记：一键分享笔记、支持自定义博客样式、无需操心格式调整。

三、典型使用场景

  • 个人知识管理 (PKM)：在有10万条笔记的时候，依旧能保持比较好的可用性与性能。

  • 项目/任务管理：用属性系统跟踪状态、deadline

  • 研究与写作：资料收集→ 草稿 → 成文，层级+链接双管齐下

  • 日记/日志：内置日记模式，自动生成每日页面

为什么要选择Trilium Notes

  • 曾经我测试过使用思维导图软件构建大量节点，但使用时会明显异常的卡顿；
  • 前些年我也测试过使用知识图谱软件来构建大量节点，但实际体验给我个人的感觉是：这知识图谱除了好看，明显不是给人分享笔记、传播知识用的。
  • 记笔记、搭建个人知识库是我们当下值得一做的，搭建个人博客其实就是个人笔记的输出。
  • 传统的笔记与博客实际上是处于一种分离状态，本质上是把自己的笔记复制一份并且部署到博客平台而已，导致博客与笔记割裂感强；
  • 另外修改笔记是一件常有的事情，调整博文排版格式也是常有的事，同时博文图片管理也是个难题。
  • 直到今年接触到Trilium Notes ! 测试后，没错，这就是我一直想要的工具。

 

基于 trilium 的博客发布方案是我找到的最满意的博客发布方案，与其他博客发布方案相比有以下几个优点：

  1. 笔记与博文无缝衔接
  2. 将知识输出环节融入知识管理流程中，无需其他发布工具
  3. 图片管理方便
  4. 博文的发布、修改和下架十分方便

总结的话，Trilium Notes开源、数据主权、性能、可玩性全部拉满，十万级笔记不卡顿。放最便宜的 1 核 1G 云服务器即可。
